std.worker — run blocking work off the loop thread, deliver the result back on it.

---- raw externs (std/worker/aether_worker.c) ---- `fn` lowers to the compiler's _AeClosure ({fn, env}); the C side stores it and invokes it (on the worker thread for `work`, on the loop thread for `done`) with the emitted fn(env, args...) convention. `ptr` job handles are opaque WorkerJob*.

run fn

run(work, done) -> bool

work: fn, done: fn

Run `work` on a background thread; when it returns a `ptr` result, deliver that result to `done` on the loop thread (through the installed poster, or the drain queue if none). Returns true if the worker launched, false on failure (no threads / out of memory) — on false neither closure runs.

run_detached fn

run_detached(work) -> bool

work: fn

Fire-and-forget: run `work` off-thread and discard its result. No post-back, nothing to drain. Returns true if launched.

set_main_poster fn

set_main_poster(poster)

poster: fn

Install the host's "post to the loop thread" trampoline. `poster` is invoked (on the worker thread) with an opaque job `ptr`; the host must marshal that ptr onto its loop thread and there call `deliver(job)`. Call once at startup.

deliver fn

deliver(job)

job: ptr

Run a job's completion now, on the calling thread: invokes its `done` with the work result, then reclaims the job. The host calls this from its loop thread inside the marshaled poster callback.

drain fn

drain(max) -> int

max: int

Pump ready completions on the calling thread. `max` <= 0 runs all currently ready; a positive `max` caps it. Returns how many ran. Only jobs that finished while NO poster was installed appear here. The headless / test loop.

pending fn

pending() -> int

Launched jobs not yet delivered (running, ready-but-undrained, or in flight through the poster). For backpressure and test synchronisation.

pool_size fn

pool_size(n)

n: int

Set the bounded pool size. Only takes effect before the first run(); the default is derived from the core count. N concurrently-blocking jobs queue job N+1 (the standard SwingWorker-style trade); use run_detached for a job that must never queue.

pool_shutdown fn

pool_shutdown()

Join and free the pool threads once they finish their current + queued jobs. For deterministic teardown; NOT run automatically. Process exit abandons in-flight/queued jobs (instant, like the pre-pool model), so a job blocked in user work can never hang exit. Call this only when jobs are known to finish.
