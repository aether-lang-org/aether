new fn

new(cap_hint) -> ptr

cap_hint: int

---- Aether-side wrappers ---- Allocate a new builder with at least `cap_hint` bytes reserved. If `cap_hint` is <= 0, a small default is used. Returns null on OOM or cap exceeded. Initial logical length is 0.

append fn

append(b, s) -> int

b: ptr, s: string

Append a string. Length is taken from the AetherString header if present, else via strlen — so a plain `const char*` containing embedded NULs truncates at the first NUL. For binary-safe content, use `append_n` with an explicit length. Returns 1 on success, 0 on failure.

append_n fn

append_n(b, s, n) -> int

b: ptr, s: string, n: int

Append exactly `n` bytes from `s`. Binary-safe — embedded NULs round-trip cleanly. Returns 1 on success, 0 on failure.

append_byte fn

append_byte(b, c) -> int

b: ptr, c: int

Append a single byte (low 8 bits of `c`). Useful for separators — '\n' (10), ',' (44), 0x02, etc. — without the round-trip through a 1-byte string. Returns 1 on success, 0 on failure.

append_int fn

append_int(b, v) -> int

b: ptr, v: int

Append the decimal representation of `v` (no padding, leading '-' on negatives). Avoids constructing an intermediate AetherString from string.from_int. Returns 1 on success, 0 on failure.

append_long fn

append_long(b, v) -> int

b: ptr, v: long

Append the decimal representation of a 64-bit `v`. Use for IDs, timestamps, and offsets that overflow 32 bits. Returns 1 on success, 0 on failure.

append_hex fn

append_hex(b, v, width) -> int

b: ptr, v: long, width: int

Append `v` as lowercase hexadecimal. `width` is a minimum nibble count — 0 for natural width, >0 to zero-pad; a value wider than `width` grows rather than truncating. Negative `v` is encoded as its unsigned bit pattern. Returns 1 on success, 0 on failure.

append_codepoint fn

append_codepoint(b, cp) -> int

b: ptr, cp: int

UTF-8 encode the Unicode code point `cp` and append it (1–4 bytes). Returns 0 — appending nothing — for invalid code points (negative, surrogate halves 0xD800–0xDFFF, or above 0x10FFFF); 1 on success.

length fn

length(b) -> int

b: ptr

Current accumulated length in bytes. Returns -1 if `b` is null.

capacity fn

capacity(b) -> int

b: ptr

Current buffer capacity in bytes. Companion to `length` — useful for confirming a `reserve` hint took or observing growth. Returns -1 if `b` is null.

reserve fn

reserve(b, additional) -> int

b: ptr, additional: int

Pre-grow the buffer so the next `additional` bytes of appends do not reallocate. A performance hint only — the builder grows on demand regardless. `additional` counts from the current length. Returns 1 on success, 0 on failure.

truncate fn

truncate(b, new_length) -> int

b: ptr, new_length: int

Pop accumulated content back to `new_length` bytes. Only shrinks — a `new_length` past the current length is rejected. The buffer is retained, so a truncate-then-append reuse loop keeps the growth amortisation. Returns 1 on success, 0 on failure (null builder, out-of-range `new_length`).

clear fn

clear(b) -> int

b: ptr

Reset accumulated content to empty (truncate to 0). The buffer is retained for reuse across loop iterations. Returns 1 on success, 0 if `b` is null.

finish fn

finish(b) -> string

b: ptr

Finalise into a refcounted AetherString and destroy the builder. After this call, `b` is invalid — do not call append, length, or free on it. The returned string is heap-owned by the caller; the codegen's heap-string-tracker reclaims the buffer at scope exit.

finish_with_length fn

finish_with_length(b) -> (ptr, int)

b: ptr

Binary-safe finalise: returns the raw data buffer and its exact byte length as `(ptr, int)`, with NO NUL terminator appended — embedded NULs are preserved as content. Use this for binary protocol assembly (CBOR, msgpack, frames, packed records) where the v1 `finish`'s NUL-terminated `string` would be truncated by a strlen-based consumer downstream. The caller owns the returned pointer and MUST free it with a raw libc free (e.g. via `mem`-style handling) — position 0 is `ptr`, so it is not codegen-heap-tracked. An empty builder yields a non-null buffer with length 0. After this call `b` is invalid.

free fn

free(b)

b: ptr

Discard the builder without producing a string. Idempotent on null. Use on error paths where the in-progress output is no longer wanted.
