# Code of Conduct

## Be Nice

No harassment or hate speech of any kind will be tolerated

## Stay On Topic

This repo is about Aether language development. Conversations in issues, pull reqests and other forums should focus 
on the language, related projects in the GitHub org, and (less so) projects using Aether elsewhere. Try to find a way 
back from side conversations.

## Be Mature

- Use "I could be wrong, but..." when debating
- Criticize things, not people: "Could this have tests, please" vs "You should have written tests" (and more strongly attacking varients)
- Assume good intentions
- Value learning over being right
- Park your ego please, and do so regardless of whether your account/presence is in your own name, a pseudonym/nick, or a fully anon account

## Like LLM contributions and love tests

- Go somewhere else to bash developers who use LLMs for coding
- Tests: Appreciate small, medium, large, unit/spec, perf/load. 
- Also appreciate component/service and ui-clicking (both integration classes) where applicable
- Accept that you may be asked to add more tests to your pull request or make the ones you have better
- Don't complain if we reverse your pull-request back to an intention and work from there (see [CONTRIBUTING.md](CONTRIBUTING.md))

## Enforcement

- Violations get a private warning first - at least if its in the personality/character/values realm
- Repeated or severe violations result in exclusion
- Decisions can be appealed by contacting maintainers, with Nic being the benevolant dictator if needed
